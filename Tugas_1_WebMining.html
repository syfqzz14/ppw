#📖Crawling

Awalnya saya ingin mengambil data jurnal dari Springer Nature tanpa harus mencarinya secara manual di website. Untuk itu saya menggunakan API yang disediakan Springer. Supaya bisa mengaksesnya, saya terlebih dahulu menyiapkan API key yang sudah didaftarkan di situs Springer Developer. API key ini nantinya berfungsi seperti kunci masuk agar permintaan saya diterima oleh server Springer.

Di dalam program, saya menggunakan tiga buah pustaka. Requests saya gunakan untuk mengirim permintaan ke API Springer. Pandas saya gunakan untuk mengubah hasil yang dikembalikan API ke bentuk tabel sehingga lebih mudah diolah. Terakhir, karena saya menjalankan ini di Google Colab, saya juga menggunakan files agar file yang sudah jadi bisa langsung saya unduh ke komputer.

Setelah itu saya tentukan kata kunci yang ingin saya cari. Saya memilih beberapa topik yang berhubungan dengan data mining, seperti web mining, web usage mining, dan data mining. Kata kunci ini akan saya kirim ke API Springer agar sistem mengembalikan daftar artikel yang relevan.

Proses pencarian berjalan berulang-ulang untuk setiap kata kunci. Jadi, misalnya saya kirim kata kunci “web mining”, API Springer akan membalas dengan sekumpulan data artikel dalam format JSON. Dari sana saya cek apakah permintaan berhasil atau tidak. Kalau berhasil, saya ambil data yang penting-penting saja: judul artikel, DOI, nama jurnal penerbit, abstrak, serta link ke artikelnya. Semua data tersebut saya simpan ke dalam sebuah list supaya bisa dikumpulkan dari berbagai kata kunci.

Setelah semua kata kunci selesai diproses, saya ubah daftar hasil pencarian tadi menjadi tabel menggunakan pandas. Kemudian saya simpan tabel itu ke dalam sebuah file CSV dengan nama springer_crawling.csv. File CSV ini dipilih karena formatnya sederhana dan bisa dibuka di Excel atau Google Sheets.

Bagian terakhir, karena saya bekerja di Google Colab, saya tambahkan perintah files.download(). Dengan begitu, setelah program selesai dijalankan, langsung muncul tombol untuk mengunduh file CSV hasil crawling. Jadi saya bisa langsung menyimpan file tersebut ke komputer tanpa repot.

Secara keseluruhan, alurnya sederhana: saya hubungkan ke API Springer, kirim kata kunci, ambil informasi artikel yang relevan, simpan semuanya ke CSV, lalu unduh hasilnya. Dengan cara ini, proses mencari referensi jurnal bisa lebih cepat dan praktis dibandingkan harus membuka situs Springer dan mencari satu per satu.

import requests
import pandas as pd
from google.colab import files  # untuk download CSV

# API Springer
API_KEY = "99d0b72ecaa3adcb617eb084ed046cf2"
URL = "http://api.springernature.com/meta/v2/json"

# Kata kunci pencarian
keywords = ["web mining", "web usage mining", "data mining"]

# List untuk menampung semua hasil
all_records = []

for keyword in keywords:
    params = {
        "q": keyword,
        "api_key": API_KEY,
        "p": 20   # jumlah record per query
    }

    response = requests.get(URL, params=params)

    if response.status_code == 200:
        data = response.json()
        print(f"Kata kunci '{keyword}' - total hasil: {data['result'][0]['total']}")

        for record in data["records"]:
            all_records.append({
                "keyword": keyword,
                "doi": record.get("doi", "N/A"),
                "title": record.get("title", "No title"),
                "publicationName": record.get("publicationName", "N/A"),
                "abstract": record.get("abstract", "No abstract"),
                "url": record.get("url", [{}])[0].get("value", "N/A")
            })
    else:
        print("Error:", response.status_code, response.text)

# Simpan ke CSV
df = pd.DataFrame(all_records)
csv_filename = "springer_crawling.csv"
df.to_csv(csv_filename, index=False, encoding="utf-8-sig")

print("\nData berhasil disimpan ke springer_crawling.csv")

# 🔽 Download langsung di Colab
files.download(csv_filename)
